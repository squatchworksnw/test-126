# Controlled Human Pilot Testing Packet

Purpose: validate whether real people can use SquatchWorks for daily operations with minimal training.

This is not a demo. The goal is to observe what the screen communicates on its own.

## Facilitator Opening

Thank you for helping test SquatchWorks. We are testing the system, not you. If something is confusing, that is useful feedback.

Please think out loud. I will only help if you are stuck for 60 seconds.

Use the app as naturally as possible. Do not worry about making mistakes.

## Trust And Clarity Watchlist

During every task, listen for these exact trust signals:

- "Did it go through?"
- "Where did it go?"
- "What does that mean?"
- "I don't know which one to click."

When one occurs, record the exact screen, label, and action the tester was looking at. Note whether a success confirmation, status label, empty state, or button hierarchy caused the uncertainty.

## Success Criteria

Use these result labels:

- Pass: user completes task without help.
- Assisted pass: user completes task after one hint.
- Fail: user cannot complete task or needs step-by-step help.

Confidence rating after each task:

- 1: I could not do this alone.
- 2: I would need help.
- 3: I might need help.
- 4: I could probably do it.
- 5: I could do this confidently.

Target:

- Average confidence 4 or higher for core daily tasks.
- No role-critical task below 3.
- No task averages more than 1 help request.

## What To Capture

Capture exact wording whenever possible.

Confusion notes:

- What word or screen confused the tester?
- Did they open the wrong screen?
- Did they stop for more than 10 seconds?
- Did they ask, "Where is..." or "Did it save?"
- Did they ask, "Did it go through?" or "Where did it go?"
- Did they say, "What does that mean?" or "I don't know which one to click?"

Accessibility notes:

- Text too small?
- Tap target too small?
- Too much reading?
- Hard to know what changed?
- Status depends on color only?

Emotional response notes:

- Calm
- Confident
- Unsure
- Frustrated
- Afraid of making a mistake
- Relieved after confirmation

## Severity Guide

- P0: Blocks task completion or creates risk of lost operational data.
- P1: User can finish, but confusion or help is likely.
- P2: Polish, wording, layout, or training improvement.
